研究背景

由原简报的核心问题、选取理由和相关性整理，不替代原文背景综述。

长序列注意力的二次复杂度会在高频聚变诊断和长窗口事件流中迅速耗尽显存。已有线性化方法很多，但如果不知道哪些状态更新设计保留 softmax 的关键行为，迁移到诊断任务时容易只得到速度而损失罕见前兆召回。 从本日报的选取理由看，新增 / arXiv 2026-07-08。它不只是提出一个线性注意力替代，而是在 frozen-backbone 条件下拆解 softmax 的 key-dependent rank-1 orthogonal projections，并给出 sink token、短卷积和固定预算 cache routing，直接对应 P1 的长序列显存瓶颈。

核心问题

如何在冻结原始 Transformer 主干或尽量少改结构的条件下，把 causal self-attention 线性化，同时保留长上下文检索和下游精度？ 对当前主题的关键意义在于：最小实验是在现有诊断长窗口模型上比较 softmax attention、线性 attention、Mamba/DeltaNet 类状态更新和固定预算 cache routing，在相同显存下报告 AUC、提前量、长间隔事件召回和吞吐。

方法与关键创新

论文分析 softmax attention 中 key-dependent、rank-1 orthogonal projection 的作用，指出纯门控累积的近似误差来源，并引入 sink tokens、short convolutions 与 fixed-budget cache routing 来缩小线性化差距。 阅读时需要把方法设计与主要结果一起看：摘要报告该方法扩展到 LLaMA 和 Qwen 最高 32B 模型，在 MMLU 上超过既有 post hoc baselines，并匹配复杂 adaptive-caching 框架的长上下文检索能力。

实验结果

摘要报告该方法扩展到 LLaMA 和 Qwen 最高 32B 模型，在 MMLU 上超过既有 post hoc baselines，并匹配复杂 adaptive-caching 框架的长上下文检索能力。

局限性

实验对象是语言模型，不直接处理真实时间间隔、异步多通道或诊断噪声；P1 迁移时仍需额外加入 timestamp-aware 编码和高低频通道交互。

总结

研究背景

由原简报的核心问题、选取理由和相关性整理，不替代原文背景综述。

聚变 shot 中的缺失并不总能靠邻近窗口平滑恢复，尤其是低频控制量、高频诊断突变和罕见破裂前兆。若只依赖 corrupted input 的局部上下文，模型会在非平稳阶段错过历史相似工况。 从本日报的选取理由看，新增 / arXiv 2026-07-08。它把 corrupted query 与完整历史候选在 latent space 对齐，并允许历史 embedding 预计算缓存；对聚变稀疏诊断、非平稳 shot 片段和罕见前兆的补全很实用。

核心问题

如何在缺失模式不一致时利用完整历史片段，而不让 corrupted query 与 complete candidate 的表示分布错位破坏检索质量？ 对当前主题的关键意义在于：最小实验是在 EAST/KSTAR 诊断数据中构建历史片段库，对随机缺失、整通道缺失和高频稀疏采样分别测试 ALER-TI + 现有 backbone 的插补误差、下游破裂 AUC 和检索延迟。

方法与关键创新

ALER-TI 提出 retrieval-augmented imputation 框架，核心是 Latent Embedding Alignment。它在 latent space 做 post-hoc masking，使候选 embedding 对齐 query 的 missingness pattern，同时保留历史 embedding 预计算和缓存能力，并能接入不同插补 backbone。

实验结果

摘要报告在 6 个真实数据集、不同缺失率下稳定提升强基线，并增强多种 imputation setting 的鲁棒性。

局限性

论文目标是插补，不是端到端破裂预测；历史库若包含装置状态漂移或错误标签，检索增强可能引入过时模式。

总结

这篇适合作为“历史 shot 相似片段检索 + 缺失通道补全”的工程路线。

研究相关性与启发
