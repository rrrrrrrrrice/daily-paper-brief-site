研究背景

现有 Transformer 时序基础模型能做跨变量依赖，但长上下文通常有二次复杂度，且新观测到来时需要重算完整历史。聚变高频诊断和工程量低频通道都更接近在线流式系统，不适合每个时刻重新处理全窗口。 从本日报的选取理由看，非当日新增 / 回看推荐。它把多变量预测、未来已知协变量、严格因果目标变量和 streaming constant per-patch cost 放在同一个 TSFM 里，对 P1 的高频诊断在线压缩和低频通道融合最直接。

核心问题

如何让一个时序基础模型同时支持多变量、未来已知协变量、严格因果目标变量和任意长度 streaming，而不让显存随上下文线性或二次爆炸？ 对当前主题的关键意义在于：可先把现有多诊断数据重采样为 patch stream，比较 TiRex-2 式 recurrent encoder 与 Transformer/Mamba 在相同预测提前量下的峰值显存、吞吐、AUC 和跨 shot 泛化。最小实验是在现有破裂或异常检测数据上只替换时序 backbone，不改标签和训练协议。

方法与关键创新

论文使用 recurrent xLSTM 架构，组合 bidirectional time mixer 与 asymmetric grouped-attention variate mixer。时间维通过递归记忆保持 constant per-patch inference，变量维通过分组注意力融合未来已知协变量，同时对目标变量保持因果约束。 阅读时需要把方法设计与主要结果一起看：摘要报告 TiRex-2 在 GIFT-Eval 与 fev-bench 上达到 state-of-the-art zero-shot performance，stream 到任意上下文长度时保持稳定，并在单变量模式使用 38.4M active parameters，多变量预测额外激活 44.1M 参数。

实验结果

摘要报告 TiRex-2 在 GIFT-Eval 与 fev-bench 上达到 state-of-the-art zero-shot performance，stream 到任意上下文长度时保持稳定，并在单变量模式使用 38.4M active parameters，多变量预测额外激活 44.1M 参数。

局限性

论文摘要未显示对真实不规则时间戳、缺失非随机机制或聚变高频 MHz 级采样的直接实验；未来已知协变量在聚变控制中可能受执行器延迟和反馈策略改变影响。

总结

TiRex-2 是本轮 P1 最值得读的工程基线，重点看它如何把 streaming、跨变量融合和因果约束放进同一架构。

许多科学诊断都不是完整规则网格。无线信道图重建中的稀疏 pilot 与聚变诊断中的稀疏测点、坏点、遮挡和局部可见光证据类似，关键是先尊重观测支撑，再引入结构先验。 从本日报的选取理由看，非当日新增 / 回看推荐。它明确指出零填充会把采样 mask 与真实场混在一起，先做 support-aware 表征再做 Fourier refinement 和 heteroscedastic prediction；这对稀疏诊断、缺失机制和可见光局部证据重建都有价值。

核心问题

如何从极低覆盖率的稀疏观测中重建连续场，同时避免零填充造成采样 mask 与真实物理场混叠？ 对当前主题的关键意义在于：可把 ECEI/BES/可见光局部 patch 或稀疏磁探针映射成带 mask 的空间或时空场，比较零填充卷积、partial convolution、Anchor-CKM 式 Fourier refinement 在缺失率变化下的重建误差和下游破裂预测性能。最小实验是在人为 mask 的既有诊断网格上做重建再接原破裂分类器。

方法与关键创新

Anchor-CKM 先用 support-aware partial convolutions 形成 pilot-supported representation，再执行 layout-conditioned dual-path Fourier refinement，最后用 coordinate-based heteroscedastic prediction 输出均值与逐位置预测方差。

实验结果

摘要报告在 DeepMIMO transmitter-disjoint 场景、missing ratio 0.3 到 0.95、甚至 5% 到 10% pilot coverage 下，相对最强复现基线降低 received-power RMSE 0.79 到 1.33 dB；消融显示 pilot-support stabilization 贡献最大。

局限性

原始任务是无线场重建，不是时间序列预测；layout prior 在聚变中要替换为磁面、探针几何或相机标定，且 Fourier refinement 对突发非平稳事件可能过平滑。

总结

它提供了一个清晰的“先观测支撑、后结构先验”的稀疏诊断重建范式。

P1 需要线性内存模型处理高频长序列，但破裂前兆、异常控制动作或罕见 MHD 活动往往是少数关键事件。完全压进固定状态会提高吞吐，却可能牺牲 needle recall。 从本日报的选取理由看，非当日新增 / 回看推荐。线性注意力和 SSM 的固定状态会遗忘少数关键关联，HOLA 用预测残差触发有界 KV cache 写入；可迁移到破裂预警中“少数前兆不能被压缩状态覆盖”的问题。

核心问题

如何在保持线性注意力低内存优势的同时，为无法线性压缩的关键关联保留可精确读取的有界记忆？ 对当前主题的关键意义在于：可在 Mamba/线性 attention 诊断模型旁加入小容量 event cache，写入高预测残差、高不确定性或高物理风险 token，比较无 cache、recency cache 和 residual-triggered cache 的提前预警、漏报率与显存。最小实验是冻结现有 encoder，只加一个可检索事件库和轻量融合头。

方法与关键创新

HOLA 保留 delta-rule recurrent state 作为压缩记忆，并增加 bounded exact KV cache。cache 不使用学习驱逐模块，而是保留具有较大 beta * residual norm 的 token，再用解耦 RMSNorm-gamma cache read 做尖锐检索。 阅读时需要把方法设计与主要结果一起看：摘要报告 340M 参数模型在 15B SlimPajama tokens 上训练后，Wikitext perplexity 从 27.32 降至 22.92，并在 RULER needle-in-a-haystack recall 上对 32k tokens 保持更强鲁棒性。

实验结果

摘要报告 340M 参数模型在 15B SlimPajama tokens 上训练后，Wikitext perplexity 从 27.32 降至 22.92，并在 RULER needle-in-a-haystack recall 上对 32k tokens 保持更强鲁棒性。

局限性

论文验证对象是语言模型，不是连续时间诊断；KV cache 会带来额外显存，写入规则是否能对应物理前兆仍需重新定义。

总结

HOLA 对 P1 的价值不是语言结果，而是“压缩状态 + 稀有事件精确记忆”的混合记忆设计。

研究背景

由原简报的核心问题、选取理由和相关性整理，不替代原文背景综述。

业务流程 OCEL 与聚变多诊断看似跨域，但都有“一个事件同时关联多个对象或通道”的结构。直接 pairwise graph 或 flatten 会丢掉共享事件上下文，也容易在长序列上显存过高。 从本日报的选取理由看，非当日新增 / 回看推荐。它把多对象共享事件、对象状态变化、事件间隔和全局原型放进双流超图模型，并报告最高 24 倍峰值显存下降；对异步多通道诊断的事件化表征很有启发。

核心问题

如何在多对象事件日志中保留事件与对象的高阶关系，同时捕捉局部状态变化和全局演化模式？ 对当前主题的关键意义在于：可把 shot 内的控制动作、阈值越界、峰值检测、缺失段和图像异常区域转成 hyperedge 事件，通道或诊断子系统作为对象，比较 flatten Transformer、temporal graph 和 EHHN 式 hypergraph 在显存与提前预警上的差异。最小实验是只用离散事件流训练 next-risk 或 next-event 预测头。

方法与关键创新

EHHN 将 prediction prefix 表示为 heterogeneous hypergraph，用 event-object hyperedges 绑定共参与对象，用 lifecycle hyperedge 表示主对象已观测生命周期。模型采用 micro-spatial stream 建模事件驱动对象状态演化，macro-evolution stream 用检索全局原型捕捉时间动态。

实验结果

摘要报告 EHHN 在 4 个公开 OCEL benchmark 上相对 9 个基线取得最佳 accuracy 和 macro F1，较最强基线分别提升最高 8.1 和 12.4 个百分点；相对最强 OCEL-native graph baseline 峰值 GPU 显存最高降低 24 倍。

局限性

原始数据是离散业务事件，不处理连续波形和真实 Δt embedding；把聚变信号转成事件需要阈值、峰值或学习型 tokenizer，可能损失幅值细节。

总结

EHHN 是 P1 的事件化建模备选路线，尤其适合稀疏报警、控制动作、诊断状态切换与多对象关联。
