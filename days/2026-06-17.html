核心问题

当前 TSQA 与时序 agent 工作大多默认输入是规则采样序列，但真实系统中的时序数据往往异步、缺测且缺失机制本身含信息，导致现有评测与现实需求脱节。 对当前主题的关键意义在于：如果你在做托卡马克诊断信号理解、异步多通道日志分析、LLM+工具的实验数据问答，这篇很相关。它解决的是“如何系统评估模型是否真的懂不规则时序”，而不是“如何把 MSE 再降一点”。

方法与关键创新

提出 IRTS-ToolBench，覆盖 13 个领域、10 类任务、1700 个问题，把不规则时序推理拆成三层能力：标准推理、 irregularity-specific reasoning、 regularity-irregularity interface reasoning。核心新增有三点： 1. 用 LLM 引导“语义合理”的 regular-to-irregular 变换，而不是简单随机 dropout； 2. 提供 30 个工具的工具库，包括 irregularity operator 和分析工具； 3. 用多模型共识生成 golden tool set，支持评测 LLM 与 agent 的工具调用行为。

实验结果

论文不只是建 benchmark，也做了零样本评测。作者报告 Qwen3.6-27B 的总体准确率达到 78.59；Claude-Opus-4.7 在不同设置下总体约 74-77%。工具调用对某些不规则任务有明显帮助，例如 DeepSeek-V4-Flash 在 irregularity severity estimation 上从 31.33 提升到 98.67，regularity recovery 从 64.67 提升到 89.33。这说明当前模型对“不规则性本身”的理解仍然不稳，但工具增强确实有效。

局限性

这是 benchmark/agent 评测论文，不是直接提升预测精度的方法；目前以单变量不规则时序问答为主，对多变量科学诊断数据的覆盖仍有限；问题与答案生成大量依赖 LLM 质量控制，未来仍需更多人工与跨社区验证。

研究背景

由原简报的核心问题、选取理由和相关性整理，不替代原文背景综述。

许多 physics-informed forecasting 方法只约束输出层预测值是否满足物理规律，但生成这些预测的隐藏状态可能依旧缺乏物理结构，长时程时误差容易在 latent transition 中累积。 从本日报的选取理由看，很多“physics-informed forecasting”只在预测输出上加约束，这篇更进一步，直接约束 latent state 和 latent transition，对长期稳定建模更有启发。

核心问题

许多 physics-informed forecasting 方法只约束输出层预测值是否满足物理规律，但生成这些预测的隐藏状态可能依旧缺乏物理结构，长时程时误差容易在 latent transition 中累积。 对当前主题的关键意义在于：对聚变领域尤其有启发，因为很多托卡马克建模任务都不是“纯统计序列”，而是带明确物理状态变量、耦合关系和守恒结构的动态系统。若你在想“怎样把物理先验放进 latent state，而不是只放进输出损失”，这篇值得精读。

方法与关键创新

提出 Phys-JEPA，把 JEPA 风格的 latent prediction 用到多变量时序预测中，并把 latent state 分解为 physical component + residual component。模型不只约束静态物理状态对齐，还约束未来 latent physical state 的动态转移一致性，相当于把 physics regularization 从 output space 推进到 predictive latent state space。

实验结果

在 Jena Climate 上，H=24 的 aggregate MSE 从 0.12482 降到 0.12273，温度 MSE 从 0.01892 降到 0.01831；在 Traffic 上，H=192 的 MSE 从 0.800784 降到 0.773873；在 Electricity 上，不同 horizon 最优变体不同，但 H=192 上完整 Phys-JEPA 取得最佳 aggregate 和 target-variable MSE。

局限性

结果提升是稳定但不夸张的“实改进”，不是碾压式领先；实验对象仍以气候、电力、交通等弱到中等物理约束任务为主，离聚变这类强耦合高维动力系统还有距离；作者也明确承认下一步需要更强物理约束问题来验证。

总结

在 Jena Climate 上，H=24 的 aggregate MSE 从 0.12482 降到 0.12273，温度 MSE 从 0.01892 降到 0.01831；在 Traffic 上，H=192 的 MSE 从 0.800784 降到 0.773873；在 Electricity 上，不同 horizon 最优变体不同，但 H=192 上完整 Phys-JEPA 取得最佳 aggregate 和 target-variable MSE。

核心问题

大规模多变量概率预测通常只比较分布精度，但在强约束系统里，预测分布是否把质量放在“不可行/不安全区域” 同样重要；现有 benchmark 和指标很少把这一点作为一等公民。 对当前主题的关键意义在于：尽管不是聚变论文，但对 disruption prediction、边界预警、控制前风险评估很有借鉴价值。很多聚变 AI 工作也存在同样问题：点预测好看，但采样分布可能覆盖大量不可运行区域，这篇提供了更严肃的思考框架。

方法与关键创新

提出 PowerPhase 概率预测 benchmark，覆盖 6 个输电网络、2,000 到 36,964 个联合预测通道，并加入 Safety_mBrier、NECV、CVaR 等约束感知指标。方法上提出 PowerForge：场景式分位数预测器，含 type-specific decoding heads、跨变量组 causal bridge、anchor-delta 电压参数化和 physics regularization。

实验结果

作者指出“分布精度”和“约束安全”对模型排名并不一致，并将其命名为 safety-fidelity trade-off。在最大规模的 PEGASE 9241（36,964 通道）上，PowerForge 在 CRPS、Distortion 与安全指标上全部最优；例如 CRPS 为 0.0077，明显优于 TimePrism 的 0.0112 与 Trans-TempFlow 的 0.0224。消融显示 anchor-delta 参数化是最关键组件。

局限性

场景是电力系统而非聚变装置；模型与指标高度依赖电网电压安全语义，不可直接平移到托卡马克；此外它更强调概率分布与可行性权衡，不是通用时序 backbone 论文。

核心问题

神经动力系统对连续时间预测很有表达力，但在传统数字硬件上顺序更新代价高、能效差，难以进入强实时、低功耗场景。 对当前主题的关键意义在于：对需要边缘实时诊断、快采样信号处理、超低功耗预测的人很有参考价值。聚变场景里，高速磁探针、辐射或电流信号的本地快速预处理未来可能正需要这类路线。

方法与关键创新

提出 FerroNDS，把两类模拟原语结合起来：时间累积积分器与频率选择振荡器，并映射到多比特 ferrodiode 存内计算硬件上。128 神经元实例可完成短时傅里叶变换并预测 500 ms horizon 的周期、准周期和混沌信号。 阅读时需要把方法设计与主要结果一起看：作者报告系统实现 sub-watt 实时运行；单神经元单次推理能耗为 1.64 μJ（200 Hz） 与 0.29 μJ（10 kHz）；相对 SRAM 数字系统面积缩减 25-40 倍；单层延迟为 3.18 ms（200 Hz） 与 63.87 μs（10 kHz）。论文声称这是 ferrodiode 首次被端到端集成进类脑计算预测框架。

局限性

这更像“时序预测硬件路线证明”而不是通用算法论文；任务类型偏周期/准周期/混沌信号，对复杂异构工业时序的适用性还需验证；系统搭建门槛高，普通实验室不易快速复现。

研究相关性与启发

对需要边缘实时诊断、快采样信号处理、超低功耗预测的人很有参考价值。聚变场景里，高速磁探针、辐射或电流信号的本地快速预处理未来可能正需要这类路线。

趋势判断

1. 聚变 surrogate 的真正分水岭，正在从“单装置快”变成“跨装置可迁移”

今天这轮里最重要的聚变新增不是又一个单点替代求解器，而是 Grad-Shafranov 神经算子的跨几何迁移设定。对未来装置、装置升级、场景变形来说，这比单一 benchmark 上再降一点误差更关键。

2. physics-informed time series 正在从输出约束走向状态约束

Phys-JEPA 的贡献在于提醒我们：如果内部 latent dynamics 没有物理结构，输出端再加 loss 也可能只是“表面守规矩”。对聚变这类强动力系统问题，这个方向大概率会持续升温。

3. 不规则时序研究的一个核心瓶颈，已经变成评测与推理协议

今天新增里，不规则时序最值得注意的不是新 forecast backbone，而是 IRTS-ToolBench 这种评测设施。说明社区开始承认：如果评测假设不对，方法进步也容易跑偏。

4. 高维时序系统的“安全-保真”分离，会越来越重要

PowerForge 虽然来自电力系统，但它指出的 tension 对聚变同样存在：模型可能在 CRPS、MSE 或平均误差上不错，但一旦放到物理约束空间里，预测分布仍可能把大量质量放在危险区域。

5. 时序 AI 的部署瓶颈正在向硬件共设计转移

FerroNDS 提醒我们，下一阶段“能不能用”未必主要取决于模型结构，而可能取决于它与硬件物理过程是否天然匹配。对于未来实时诊断与边缘控制，这是个值得长期跟踪的信号。
